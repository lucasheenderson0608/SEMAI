Requisitos de sistema do ArcGIS Pro 3.2

Antes de instalar ou atualizar o ArcGIS Pro em uma máquina virtual ou física, certifique-se que seu sistema atenda aos requisitos mínimos para executá-lo. Saiba também quais recursos são recomendados para obter o melhor desempenho.

Analise seu computador para compatibilidade

Além de comparar a compatibilidade de sua máquina com os requisitos abaixo, você pode baixar um aplicativo que fará a varredura de sua máquina e gerará um relatório de compatibilidade: Verificar a capacidade do seu computador para executar ArcGIS Pro

  • A opção Desktop Experience é necessária em todas as versões do Windows Server.

Requisitos de hardware

Os requisitos mínimos, recomendados e ideais para executar oArcGIS Pro estão listados abaixo. Os requisitos mínimos oferecerão suporte ao aplicativo, mas podem não ter um bom desempenho. Os requisitos recomendados funcionam bem na maioria das situações. O melhor desempenho é obtido usando os requisitos ideais.

Item

Suportado e recomendado

CPU

Mínimo: 2 núcleos, multithreading simultâneo

O multithreading simultâneo, ou hyperthreading, de CPUs normalmente apresenta duas threads por núcleo. Uma CPU multithread de 2 núcleos terá quatro threads disponíveis para processamento, enquanto uma CPU multithread de 6 núcleos terá 12 threads disponíveis para processamento.

O Vídeo de Movimento Completo tem especificações de CPU recomendadas e mínimas mais altas. Consulte Introdução ao Vídeo de Movimento Completo para detalhes.

Recomendado: 4 núcleos

Recomendado: 10 núcleos

Plataforma

x64

Armazenamento

Mínimo: 32 GB de espaço livre

Recomendado: 32 GB ou mais espaço livre em uma unidade de estado sólido (SSD)

Memória/RAM

Mínimo: 8 GB

Recomendado: 32 GB

Favorável: 64 GB ou mais

Memória de gráficos dedicada (não compartilhada)

Recomendado: 4 GB ou mais

Se você estiver utilizando um computador notebook com GPU integrada, considere aumentar o sistema RAM para compensar pelo uso de memória compartilhada.

Cache de visualização

O cache de visualização temporário pode consumir até 32 GB de espaço, se disponível, no local de usuário selecionado. Por padrão, o cache de visualização é gravado na subpasta do perfil do usuário \Local, para que não faça roaming com o perfil do usuário se os perfis de roaming forem habilitados pelo administrador do sistema.

DirectX*

Mínimo: DirectX 11, nível de recurso 11.0, Shader Model 5.0

Recomendado: DirectX 12, nível de recurso 12.0, Shader Model 6.0

OpenGL*

Minimum: OpenGL 4.3 com as extensões ARB_clip_control e EXT_texture_compression_s3tc

Recomendado: OpenGL 4.5 com as extensões ARB_shader_draw_parameters, EXT_swap_control, EXT_texture_compression_s3tc, e EXT_texture_filter_anisotropic

Resolução da tela

Mínimo: 1024x768

Recomendado: 1080p ou superior

* Se os requisitos do DirectX não forem atendidos, oOpenGL será automaticamente usado. Você também pode alterar manualmente para OpenGL, como quando suspeitar de um problema com o driver DirectX.

Anotação:

  • Certifique-se de utilizar o driver disponível mais recente.

  • Um processador que suporte o conjunto de instruções AVX é necessário para algumas ferramentas de geoprocessamento.

  • Ao usar uma GPU, garanta que ela seja suportada pelo fabricante.

  • Serviços de armazenamento na nuvem, como Microsoft OneDrive e Google Drive, não são suportados, a menos que indicado de outra forma na documentação sobre ferramentas e funcionalidades específicas. Saiba mais sobre ArcGIS Pro e serviços de armazenamento na nuvem.

Computação de uso geral em um GPU

O suporte para computação de propósito geral em uma GPU usando CUDA não é necessário para executar o software, mas é necessário para executar a maioria das funcionalidades de aprendizagem detalhada. A GPGPU também pode acelerar algumas ferramentas do Spatial Analyst.

Item

Suportado e recomendado

Tipo de GPU

GPU NVIDIA com capacidade de computação CUDA mínima de 3.7; 6.1 ou posterior recomendado. Consulte a lista de cartões habilitados para CUDA para determinar a capacidade de computação de uma GPU ou verifique a seção CUDA Compute do verificador de requisitos do sistema.

Driver GPU

Drivers GPU NVIDIA : versão 456.38 ou posterior é necessária.

Memória gráfica dedicada

Mínimo: 6 GB

Recomendado: 16 GB ou mais

Os requisitos de memória dependem da arquitetura do modelo e do tamanho do lote que está sendo usado.

Anotação:

Um driver de GPU desatualizado fará com que as ferramentas de aprendizagem detalhada falhem com erros de tempo de execução indicando que o CUDA não está instalado ou um conjunto de ferramentas incompatível está presente. Verifique se você tem drivers de GPU atualizados, fornecidos diretamente pela NVIDIA.

Requisitos mínimos

Microsoft .NET

Microsoft .NET Desktop Runtime 6.0.5 ou uma versão de patch posterior (6.0.6 e assim por diante), usando um instalador Windows x64, é necessário. A presença do Microsoft .NET Desktop Runtime 7 ou superior é irrelevante.

Licenças

Três tipos de licença estão disponíveis para ArcGIS Pro 3.2. Dependendo do tipo que você usa, você pode precisar de software adicional ou acesso aArcGIS Online.

  • Licença de Usuário Nomeado—O ArcGIS Pro é licenciado por uma conta organizacional no ArcGIS Online ou ArcGIS Enterprise. Se a licença for através do ArcGIS Online (a opção padrão), o software de licenciamento não será exigido, pois o ArcGIS Online é um serviço baseado em nuvem fornecido pela Esri.

    Se a licença for por meio do ArcGIS Enterprise, use a versão mais recente do ArcGIS License Manager independentemente da versão do ArcGIS Enterprise usada.

  • Licença de Uso Concorrente—A versão mais recente do ArcGIS License Manager é exigida para gerenciar licenças do ArcGIS Pro 3.2.
  • Licença de Uso Único—Nenhum software adicional é exigido para licenciar.

Azure N-Series VMs

NVv4-series

  • NV4: Bom para uso 2D. Capaz de fornecer uma experiência 2D sólida com ArcGIS Pro.
  • NV8: Bom para uso 2D, como também, uso 3D leve. O mesmo que NV4, mas capaz de renderizar e mover renderização 3D com um pouco mais de suavidade.
  • NV16: Plataforma sólida para uso 3D médio a pesado. Este é um sistema que pode beneficiar muitos usuários avançados que trabalham com 3D, como também, aqueles com grandes conjuntos de dados e necessidades de análise.
  • NV32: Esta é uma boa plataforma para 3D pesado e conjuntos de dados maiores com grandes necessidades de análise.

Série NCasT4_v3

  • As máquinas virtuais da série NCasT4_v3 são equipadas com GPUs NVIDIA Tesla T4 e CPUs AMD EPYC 7V12 (Roma). As MVs apresentam até 4 GPUs NVIDIA T4 com 16 GB de memória. Ideal para usuários 3D.

Série - NVadsA10_v5

  • As máquinas virtuais da série NVadsA10v5 são equipadas com GPUs NVIDIA A10 e CPU AMD EPYC 74F3V(Milan). A série NVadsA10v5 oferece máquinas virtuais com GPUs NVIDIA parciais.

Anotação:

Ao usar ferramentas do ArcGIS Spatial Analyst ou recursos de aprendiza automáticos que usam GPU, mais de uma GPU pode ser útil para dar suporte à visualização e computação. Considere esta opção se estiver executando ferramentas do Spatial Analyst ou recursos de aprendizagem automática e a exibição tornar-se irregular. Consulte as listas atuais de ferramentas do Spatial Analyst e recursos de aprendizagem automática que utilizam a GPU para computação.
